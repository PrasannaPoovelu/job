# AI Job Application Optimizer

An AI-powered web app that turns a job description + your resume into an
ATS-friendly optimized resume, a tailored cover letter, and an interview
preparation guide — matched to a target region (USA / UK / Europe / India).

The optimized resume can be downloaded as a **styled one-page PDF** with a
navy header band, two-column skills table, blue accent links, and
right-aligned dates.

Built with **React 18 + TypeScript + Vite + Tailwind**, tested with **Vitest**,
PDF rendered with **@react-pdf/renderer**.

## Features

- Drag-and-drop upload for both Job Description and Resume (`.pdf`, `.docx`, `.txt`, `.md`)
- Region-specific resume formatting (USA, UK, Europe, India, Other)
- Three tabbed outputs: **Optimized Resume**, **Cover Letter**, **Interview Prep**
- **PDF download** of the optimized resume in a clean one-page format
- **Optimization metrics** — before/after JD match, ATS score, list of keywords added
- **AI Job Hunter** — searches real listings on Adzuna + The Muse + Remotive,
  scores every job for fit (0–100), and groups results into
  high-probability / medium-probability / stretch tiers with verified Apply links
- Copy or download each output as a `.txt` file
- **Multi-provider** — Gemini (free), Anthropic, or local Demo Mode
- One-click "Try in Demo Mode" recovery if the API hits a billing/rate-limit error
- Strict prompt contract — no fabricated experience, no keyword stuffing
- Full unit test coverage for the agent core, parser, file utilities, jobs, and UI

### About the Job Hunter

Every link in the Job Hunter results comes from a real job-board API — never
generated by the LLM. The LLM is used only for (1) reading your resume to
pick target roles and (2) batched fit-scoring against jobs the API returned.

| Source        | Auth needed                              | Cost                    | Coverage                                                           |
| ------------- | ---------------------------------------- | ----------------------- | ------------------------------------------------------------------ |
| Adzuna        | Free API key                             | Free (250 req/day)      | India + 18 countries, broad mix                                    |
| **JSearch**   | Free RapidAPI key                        | Free (200 req/month)    | **Indeed** + LinkedIn + Google for Jobs + ZipRecruiter aggregated  |
| The Muse      | None (optional key for higher rate limit)| Free                    | Mostly tech, US-focused                                            |
| Remotive      | None                                     | Free                    | Remote-only                                                        |

If you don't add any keys, the app still works — it falls back to The Muse + Remotive.

## Quick start

```bash
# 1. Install dependencies
npm install

# 2. Configure your API key (pick ONE — Gemini is free)
cp .env.example .env.local
# then edit .env.local and add VITE_GEMINI_API_KEY

# 3. Run the dev server
npm run dev
```

Open http://localhost:5173 in your browser.

## Security note

The Anthropic SDK runs in the browser using `dangerouslyAllowBrowser: true`,
which exposes your API key to anyone who loads the page. The Gemini path uses
plain fetch — same caveat applies. **For anything beyond local use, put a
small backend proxy in front of the API call** so the key stays server-side.

## Deploy to GitHub Pages

The repo ships a workflow at `.github/workflows/deploy.yml` that builds the
app on every push to `main` and publishes `dist/` to GitHub Pages.

### One-time setup

1. **Enable Pages with the Actions source.** Go to your repo →
   *Settings → Pages → Build and deployment* → set **Source = GitHub Actions**.
2. **Add your API keys as repo secrets** (optional but recommended). Go to
   *Settings → Secrets and variables → Actions → New repository secret* and add:
   - `VITE_GEMINI_API_KEY` (recommended — free tier)
   - `VITE_ADZUNA_APP_ID`, `VITE_ADZUNA_APP_KEY` (free)
   - `VITE_JSEARCH_API_KEY` (free)
   - Any of the others you want enabled in production.

   Without these, the deployed site still loads — it just falls back to Demo Mode.

### Common deployment gotchas (already handled)

- **Blank screen / 404'd assets** — Vite's `base` is set to `/job/` in production
  so asset URLs match `https://<user>.github.io/job/...`. If you fork the repo
  under a different name, set `VITE_BASE_PATH=/new-name/` in the workflow env.
- **Jekyll stripping `_chunks`** — `public/.nojekyll` opts out of Jekyll
  processing so Vite's hashed chunk files survive.
- **API keys baked into bundle** — Vite inlines `VITE_*` env vars into the JS
  at build time. Anyone who views the deployed site can read these from the
  bundle, so use rate-limited free keys (Gemini, Adzuna, JSearch). Never put a
  paid Claude/Apify key in repo secrets for a public-Pages deployment — set up
  a backend proxy instead.

### Re-deploying

Just push to `main` — the workflow runs automatically. To force a rebuild,
go to *Actions → Deploy to GitHub Pages → Run workflow*.
